# プロジェクトゴール文書 — AI開発成功事例データベース

> このドキュメントはプロジェクトの「単一の真実の源(Single Source of Truth)」です。
> 新しい作業セッションを始めるときは、まずこれを読んでから着手します。
> 決定が変わったら必ずここを更新します(古い決定を残さない)。

---

## 1. 一言でいうと(Mission)

海外の個人開発者がAIアプリ/ツールで急成長・収益化した事例を、**数字とテクニックで構造化して比較・検索できるデータベース型メディア**。
読者は「日本の個人開発者・副業層・AIプロダクトを作りたい人」。

## 2. なぜやるのか(Why / 最重要ゴール)

- **オーナー(私)の生活費を稼ぐこと**が最終ゴール。趣味でも実験でもなく、収益が出ることが成功の定義。
- そのため「完璧なサイト」より「早くローンチして有料導線を検証する」ことを優先する。

## 3. 何で差別化するか(Differentiation)

元ネタであるウマたん氏のニュースレターは「1記事ずつ読む読み物」。
本サイトの価値は彼の記事には無い **一覧性・検索性・比較可能性**:

- 人物名・アプリ名・収益・成長期間・ユーザー数・テックスタック・マーケ手法を**構造化データ**として保持
- 収益規模/期間/カテゴリ/手法タグで**フィルタ・ソート**できる
- 記事を読む前に「数字で横断比較」できるのが売り

参考になる前例: 「Onbo Hub」(売れてるアプリのオンボーディング画面を集めた有料/API提供のDB。同じニッチで課金文化が成立している)。

## 4. 確定した決定(Settled Decisions)

| 項目 | 決定 | 理由 |
|---|---|---|
| サイト形態 | データベース型メディア | 記事が増えるほど価値が積み上がる。SEOで「AI開発 個人開発 成功事例 一覧」等の検索意図に強い |
| 技術スタック | Next.js (App Router) + Tailwind CSS、Vercelデプロイ | ディレクトリ名が "AI DEV site"、Vercel系MCPツールも利用可。SEO・更新性・i18nが素直 |
| 主な見せ方 | 事例カード一覧 + フィルタ → 詳細ページ | DB型 + ブログ型のハイブリッド |
| 言語 | 日本語・英語の両対応(言語切り替え可) | 日本語=有料会員、英語=広告+広いリーチ。両方の長所を取る |
| URL設計 | 言語をパスで分離 `/ja/...` `/en/...` + `hreflang` | Cookie切替だけだとSEOで片方しかインデックスされない。両言語をSEOに乗せるには必須 |
| 翻訳 | AI翻訳でOK。ただし固有名詞・数字・タグは言語共通で構造化管理し、翻訳対象は説明文・本文のみ | 機械翻訳そのままの薄いコンテンツ判定を避け、運用コストも下げる |
| コンテンツの出自 | 元ニュースレターは**参考のみ**。丸ごと転載しない。自分の言葉/独自フォーマットで書き直す | 著作権リスク回避 + 独自の構造化が差別化 |
| 詳細ページのUI見本 | **Starter Story の事例ページ構成を手本にする**(右サイドのサマリーボックス + 本文セクション + premium ペイウォール)。例: starterstory.com/stories/dorik-breakdown | 完成度が高く、我々のデータモデルとほぼ1対1で対応する |
| Starter Story との違い | ①AIアプリ特化(鮮度) ②日本語(+英語)で空白を取る ③`painPoint`/`playbook` を**無料で前面**に出す(SSは有料に隠しがち) | 手本にするが劣化コピーにしない差別化の核 |

## 5. 収益モデル(Monetization)

1つに絞らず段階的に全部積む。優先順位:

1. **アフィリエイト(初日から仕込む)** — 各事例の「使用テックスタック」(Cursor / Claude Code / Replit / Supabase / Vercel / Resend 等)に紹介リンク。DB型は文脈が合うので強い。労力ほぼゼロ。
2. **有料会員(本命 / 生活費を支える柱)** — 無料: 事例カード一覧・基本数字・フィルタ。有料: 深掘り分析・全件比較・CSV/APIエクスポート・新着事例の先行配信。生活費レベルの安定収益はサブスクで作る。
3. **広告(後回し / おまけ)** — 最初から貼ると会員転換を下げる。SEOでPVが万単位に育ってから補助的に。

## 6. データモデル(確定)

詳細は `docs/DATA_MODEL.md`(確定版・TypeScript型と固定タグ辞書つき)。要点:

- **収益は `mrrUsd`(USD建てMRRに正規化)を主ソートキー**にして全件を1軸で比較。元表記は `revenueRaw` で別途保持。
- **タグは固定辞書(enum)で管理**(category / techStack / marketingTags)。自由入力禁止で表記ゆれを防ぐ。辞書は実データ38件から抽出済み。
- **無料/有料は1記事内で混在**。`body.free`(常に無料)と `body.premium`(深掘り分析=有料ロック)に分割。
- 数字・固有名詞・タグキーは言語共通(1回入力で日英反映)。翻訳対象は title / summary / revenueRaw / body のみ。
